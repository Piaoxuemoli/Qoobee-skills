# README Update Rules

## When to Update

Update `README.md` after any of these changes:

- Adding or removing a skill
- Adding or removing a major feature (e.g., PPT engine, new export format)
- Changing project structure or directory layout
- Adding new dependencies or setup steps
- Changing the project's public API or usage patterns

## Format

- README must be bilingual: Chinese (zh) first, English (en) second, separated by `---`.
- Use consistent heading structure across both languages.
- Keep the table of contents / overview section in sync.

## What NOT to Update

- Minor bug fixes, internal refactors, or doc-only changes do not require README updates.
- Do not add changelogs to README — use git history for that.
